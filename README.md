# Word Frequency Analyzer API

Асинхронный сервис для анализа частотности словоформ в текстовых файлах. Проект выполнен в рамках тестового задания для **Kaspersky**.

## Особенности архитектуры

Проект спроектирован с учетом обработки **огромных файлов** (GB+) и обеспечения **высокой доступности** (High Availability) при одновременных запросах от множества пользователей.

### Ключевые решения:
* **Асинхронная обработка (Celery + Redis):** Тяжелые задачи по анализу текста вынесены в фоновые воркеры. API мгновенно возвращает `task_id`, не блокируя пользователя.
* **Стриминговое сохранение (Streaming Upload):** Файлы сохраняются на диск чанками (`shutil.copyfileobj`), что предотвращает переполнение оперативной памяти (RAM) даже при загрузке многогигабайтных файлов.
* **Методология (DDD элементы):** Логика разделена на слои:
    * `api` — интерфейс взаимодействия.
    * `infrastructure` — работа с очередями и NLP-процессором.
    * `domain` — базовые сущности и статусы.
* **NLP Анализ (pymorphy3):** Используется для приведения слов к нормальной форме.
* **Оптимизация Excel:** Генерация отчета происходит в режиме `write_only`, что позволяет создавать таблицы любого размера с минимальным потреблением памяти.

## Конфигурация (config.yaml)
Система использует централизованный файл конфигурации для управления окружением. Настройки считываются при старте приложения и доступны через класс Config.

YAML
http:
  url: http://localhost:8080        # Базовый URL сервиса для формирования ссылок
broker:
  broker_url: "redis://broker:6379/0"   # Адрес Redis для отправки задач (Producer)
  backend_url: "redis://broker:6379/0"  # Адрес Redis для хранения результатов (Result Backend)
  visibility_timeout: 43200             # Время (в сек), после которого задача считается зависшей (12 часов, т.к. файлы большие)
  max_retries: 0                        # Количество автоматических повторов при ошибке в воркере

## Команды управления (Makefile)

| Команда | Описание                                                                                             |
| :--- |:-----------------------------------------------------------------------------------------------------|
| `make up` | Запустить всю систему в фоне (API + Worker + Redis)                                                  |
| `make down` | Остановить и удалить контейнеры                                                                      |
| `make gen-file` | Сгенерировать тестовый `.txt` файл на 20,000 строк, кол-во строк меняется через переменные окружения|
| `make test-e2e` | Запустить сквозной тест (загрузка -> ожидание -> проверка ссылки)                                    |
